#6. Consumer Payment Connect

Client: Kaiser, Permanente

Role: Technical Lead

Location: Pleasanton, CA


Project Description:

Consumer Payment Connect is a web application that is loosely coupled to Kaiser Membership systems via Web Services and allows direct access via self-service. The Consumer Payment Connect hosts the payment landing pages and allows a variety of other functions such as setting up recurring payments, initiated the refund process, initiating the BUID transfer process and managing the payer profile.


Responsibilities:

  • Responsible for Application Design, Development, Testing and Implementation of product.
  • Developed different modules of front-end application on Windows platform using Microsoft technologies; .NET Framework 4.5, Visual Studio 2012, C#, ASP.NET MVC 4.0, ASP.Net
  • Web API, VB.NET with SQL Server 2012 as the backend for data storage
  • Used JavaScript, jQuery, Angular.JS, CSS, HTML for re-usable of code and to make User Interface more user friendly
  • Developed Web Services for processing payments with centralized databases
  • Developed SSIS packages to perform data migration tasks across systems
  • Perform root cause analysis (RCA) and determine solutions to prevent re-occurring of production issues
  • Conducted daily Stand up SCRUM meetings for updates on various projects
  • Develop unit test cases using Microsoft Visual Studio Unit Testing framework
  • Involved in QA meetings to review against the Test Plan to ensure that the test coverage is appropriate

    • Environment:

    .NET Framework 4.5, ASP.NET MVC 4.0, ASP.Net, JavaScript, jQuery, Angular.JS, CSS, HTML, Data Power XI52, MYSQL, SQL Server, SOAP UI, and Web services (SOAP).
